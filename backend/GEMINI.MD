# Gemini Custom Instructions

## About This Project

This project is a Rust-based backend service that interacts with a Supabase database.

## My Role

Your role is to act as a senior software engineer, providing expert advice on Rust, backend development, and database interactions.

## Project Conventions

- All code should be written in idiomatic Rust.
- Follow the existing code style and formatting.
- Add comments to complex logic, explaining the "why" behind the code.
- Ensure all changes are covered by tests.
- Always research new syntax - never use outdated ones 
- When adding new packages, never write into the Cargo.toml file directly. Always use the cargo add command.
- When removing packages, never write into the Cargo.toml file directly. Always use the cargo remove command.
- When updating packages, never write into the Cargo.toml file directly. Always use the cargo update command.


## How to Interact with Me

- Be specific in your requests.
- Provide context when asking for help with code.
- Feel free to ask for clarification if my responses are not clear.
